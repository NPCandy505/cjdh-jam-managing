## 2. 이번 프로젝트에서 사용할 기술 다시 보기

### 사용할 스택

1. `Node.js`
2. `Next.js`
3. `TypeScript`
4. `SQLite`
5. `Prisma`
6. `Vercel`

### 각 스택이 하는 일

1. `Node.js`
   - 자바스크립트 코드를 컴퓨터에서 실행하게 해 주는 기반
   - `npm install`, `npm run dev` 같은 명령을 실행할 때 필요하다
2. `Next.js`
   - 웹 화면과 서버 기능을 한 프로젝트 안에서 만들게 해 주는 틀
   - 페이지, API, 폼 처리, 배포 연결이 비교적 쉽다
3. `TypeScript`
   - 자바스크립트 코드에 타입 정보를 더해 실수를 줄여 준다
   - 초심자에게도 "이 값이 문자열인지 배열인지"를 빨리 파악하게 도와준다
4. `SQLite`
   - 신청곡, easy 곡 리스트, 원클릭 신청 정보를 저장하는 데이터베이스
   - 작은 프로젝트에서 시작하기 좋다
5. `Prisma`
   - 코드와 데이터베이스를 연결해 주는 도구
   - SQL을 직접 많이 쓰지 않아도 DB를 다룰 수 있게 해 준다
6. `Vercel`
   - 만든 사이트를 인터넷에 올려 배포할 때 사용하는 서비스
   - Next.js와 궁합이 좋다

## 11. 7단계: 추가 참여 멤버 입력 방식 정하기

### 목표

단순한 문자열이 아니라 `이름 + 세션` 구조로 입력받는 UI를 만든다.

### 추천 구현 순서

- [ ] 처음에는 추가 참여 멤버 입력 없이 신청곡 생성만 가능하게 만든다
- [ ] 그다음 별도 UI로 "해당 곡에 참여 멤버 추가" 기능을 붙인다

### 이유

1. 신청곡 생성과 추가 참여 멤버 수정은 사용 흐름이 조금 다르다
2. 한 번에 다 만들면 상태 관리가 복잡해질 수 있다

### 할 일

- [ ] 각 곡 행 또는 곡 상세 UI에 `추가 참여` 버튼을 둔다
- [ ] 버튼 클릭 시 이름과 세션을 입력할 수 있게 한다
- [ ] 세션은 동일하게 고정 목록 드롭다운을 사용한다
- [ ] 추가 후 표에 `홍길동(베이스)`처럼 보이게 한다

### 완료 기준

- [ ] 특정 곡에 추가 참여 멤버를 붙일 수 있다

## 14. 10단계: Prisma와 SQLite 연결

### 목표

더미 데이터가 아닌 실제 저장 기능을 붙인다.

### 초심자 메모

1. `SQLite`는 실제 데이터를 저장하는 파일 기반 DB다
2. `Prisma`는 코드로 DB를 다루기 쉽게 해 준다
3. 먼저 화면을 만든 뒤 DB를 붙이는 순서가 초심자에게 훨씬 편하다

### 할 일

- [ ] Prisma를 설치한다

```powershell
npm install prisma @prisma/client
```

- [ ] Prisma 초기화를 실행한다

```powershell
npx prisma init
```

- [ ] `schema.prisma` 파일을 열어 `SQLite`를 사용하도록 설정한다
- [ ] `JamEntry`, `AdditionalParticipant`, `EasySong`, `OneClickRequest`에 맞는 모델을 설계한다
- [ ] `.env` 파일에서 SQLite 경로를 확인한다
- [ ] 마이그레이션을 실행한다

```powershell
npx prisma migrate dev --name init
```

### 권장 주의점

- [ ] `AdditionalParticipant`는 별도 테이블로 분리하는 편이 좋다
- [ ] `EasySong`도 DB 테이블로 둘지, 초기에는 정적 데이터로 둘지 결정한다
- [ ] 초반 MVP라면 `EasySong`은 정적 데이터로 시작해도 충분하다

- [ ] 삭제 후 목록에서 사라지게 한다

### 완료 기준

## 24. 마지막 체크

개발을 시작할 때 가장 중요한 원칙은 다음 3가지다.

- [ ] 한 번에 다 만들려고 하지 말고, 보이는 화면부터 조금씩 완성한다
- [ ] 더미 데이터로 먼저 확인한 뒤 실제 DB를 붙인다
- [ ] 기능 하나를 만들 때마다 바로 실행해서 확인한다

이 문서를 따라가면, 최소한 MVP 기준의 잼 데이 관리 사이트를 단계적으로 구현할 수 있다.
